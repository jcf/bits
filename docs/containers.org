#+title: Containers

* Overview
We build all container images with Nix using [[https://code.invetica.team/jcf/nix2container][nix2container]]. This produces
OCI-compliant images — not Docker-specific — that any container runtime
supporting the OCI image specification can load (Docker, Podman, containerd,
etc.).

Key properties:

- *Reproducible* — identical inputs always produce identical images, byte for
  byte. No layer cache invalidation surprises.
- *Minimal* — we include only explicitly declared dependencies. No leftover
  package manager caches, documentation, or shell utilities.
- *No Dockerfile* — Nix expressions declare each image, composing precise
  package references rather than imperative shell commands.
- *Shared base* — =pkgs/container-base/default.nix= provides system libraries,
  user config, and OCI labels shared across all images.
- *Native ARM64* — we target =aarch64-linux=, running natively on Apple Silicon
  via OrbStack with no emulation.
- *Single-layer* — each image ships as a single OCI layer, keeping manifest
  overhead low and pulls fast.

* Images
#+begin_src sh :results output table :colnames '("Image" "Size" "Arch" "Layers")
for img in $(docker images --format '{{.Repository}}' --filter 'reference=bits-*' | sort); do
  size=$(docker images "${img}:latest" --format '{{.Size}}')
  arch=$(docker inspect "${img}:latest" --format '{{.Architecture}}')
  layers=$(docker inspect "${img}:latest" --format '{{len .RootFS.Layers}}')
  printf '%s\t%s\t%s\t%s\n' "$img" "$size" "$arch" "$layers"
done
#+end_src

* Building
#+begin_src sh :eval never
just build-containers
#+end_src

Nix builds each image and =skopeo= copies it into the Docker daemon. The first
build fetches dependencies from the Nix binary cache; subsequent builds complete
instantly if inputs haven't changed.

* Security
Every service in =compose.yml= inherits a shared security baseline via YAML
anchor:

- *No new privileges* — prevents =setuid= / =setgid= escalation.
- *Drop all capabilities* — we start with zero Linux capabilities and add back
  only what a service needs (postgres gets =CHOWN=, =SETUID=, =SETGID= for its
  init sequence; Traefik gets =NET_BIND_SERVICE= for port 443).
- *Read-only root filesystem* — the container filesystem is immutable. Postgres,
  nrepl, and tailwind override this for data volumes and mounted source.
- *tmpfs at /tmp* — each container gets its own in-memory =/tmp= with =noexec=
  and =nosuid=. nrepl and tailwind override with =exec= because brotli4j and
  lightningcss extract native libraries to =/tmp= at runtime.
- *Non-root by default* — most images run as =bits= (uid 1000). Postgres starts
  as root for data directory initialisation, then drops to uid 1000 following the
  [[https://github.com/docker-library/postgres][docker-library/postgres]] pattern.

Services that need exceptions declare them explicitly — the default is locked
down.

* Orchestration
