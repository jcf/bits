* Asset Cache Busting
** Current state
- ~src/bits/assets.clj~ — Buster component (159 lines, complex)
- ~src/bits/service.clj~ line 93 — Uses ~ring/create-resource-handler~ directly
- Buster exists but *is not integrated with reitit*

** Goals
1. Simplify ~bits.assets~ — remove OTel spans, streamline API
2. Integrate with reitit's resource handling
3. Provide ~asset-path~ function for templates to generate busted URLs

** Integration approach
Two options:

*** Option A: Wrap resource handler
Create middleware that intercepts requests for busted URLs and serves the
underlying resource with cache headers.

#+begin_src clojure
;; Request: GET /app.abc123.css
;; Buster maps abc123 → app.css
;; Serve app.css with Cache-Control: immutable
#+end_src

*** Option B: Replace resource handler
Custom handler that uses Buster's lookup to serve assets.

** Template usage
#+begin_src clojure
;; In layout
[:link {:rel "stylesheet" :href (assets/path buster "/app.css")}]
;; Renders: /app.abc123.css
#+end_src

** Simplified API
#+begin_src clojure
;; Core functions only
(defn make-buster [resources]) ; → Buster
(defn path [buster asset-path]) ; → "/app.abc123.css"
(defn lookup [buster uri]) ; → {:resource ... :content-type ...}
#+end_src
