* Goal
Wire up the asset pipeline so that:
1. Cache-busted URLs work (same code path for all environments)
2. CSS changes trigger hot reload via SSE (no full page reload)
3. Immutable cache headers for busted assets

* Current State
** What exists
- ~bits.assets~ — Buster component computes BLAKE2b-128 hashes
- ~regurgitate~ — Recomputes hashes
- ~asset-path~ — Returns busted path (~/app.abc123.css~)
- ~lookup~ — Finds asset by busted URI
- Service depends on Buster but doesn't use it
- ~morph/reload-event~ — Sends SSE to trigger ~window.location.reload()~
- Morph refresh on reconnect already works (hash comparison)

** What's missing
- Asset serving middleware (routes busted URLs to resources)
- Templates don't use Buster (hardcoded ~/app.css~)
- No file watching for Tailwind output → stylesheet event

* Design
** Asset serving middleware
Intercept requests matching busted URL pattern. Serve resource with immutable
cache headers.

#+begin_src clojure
;; Request: GET /app.abc123.css
;; Lookup in Buster → finds resource
;; Respond with resource + Cache-Control: public, max-age=31536000, immutable
#+end_src

** Template integration
Inject Buster into request. Layout uses ~asset-path~ to generate busted URLs.

#+begin_src clojure
;; In middleware
(defn wrap-buster [handler buster]
  (fn [request]
    (handler (assoc request :bits.assets/buster buster))))

;; In layout
[:link {:rel "stylesheet"
        :href (assets/path (:bits.assets/buster request) "/app.css")}]
#+end_src

** Config: reload-assets?
When ~:reload-assets? true~, recompute hashes on each request. CSS changes
produce new busted URLs, browser fetches fresh copy.

#+begin_src clojure
(defn wrap-buster [handler buster reload-assets?]
  (fn [request]
    (let [buster (cond-> buster reload-assets? assets/regurgitate)]
      (handler (assoc request :bits.assets/buster buster)))))
#+end_src

Set via config in ~bits.app/read-config~:
#+begin_src clojure
:service {:reload-assets? (= "true" (env :reload-assets))
          ...}
#+end_src

** File watching (CSS hot reload)
Watch ~resources/public/app.css~. On change, send new busted URL to clients.
Browser fetches and applies new CSS without full page reload — preserves scroll
position, form state, etc.

#+begin_src clojure
;; Watcher detects app.css change
;; → regurgitate Buster to get new hash
;; → send stylesheet event with new busted URL
;; → client updates link href
;; → browser fetches fresh CSS
#+end_src

** Client: stylesheet event handler
Replace the single stylesheet link. Warn if multiple found (YAGNI guard).

#+begin_src javascript
handlers["stylesheet"] = (data) => {
  const links = document.querySelectorAll('link[rel="stylesheet"]');
  if (links.length !== 1) {
    log.warn("Expected exactly one stylesheet. Found:", links);
    return;
  }
  links[0].href = data;
};
#+end_src

** Watcher Component
Watcher is a Component using ~java.nio.file.WatchService~. Defined in
~bits.app/components~ but only started in dev (via ~dev/bits/dev.clj~ adding
it to the running system).

- Depends on Buster and Service
- On file change: regurgitate Buster, broadcast new stylesheet URL
- Uses virtual thread for watch loop

** Broadcast mechanism
Service needs a generic way to broadcast SSE events to all connected clients.
Watcher uses this to send ~stylesheet~ events with the new busted URL.
