* Goal
Resolve the current realm from the request Host header and inject into the
request map as ~:session/realm~.

* Design
** Domain model
- *Realm*: An entity with attributes from multiple namespaces (~:tenant/*~,
  ~:creator/*~) representing a tenant's public-facing identity
- *Domain*: Separate entity for uniqueness constraint, referenced by realm via
  ~:tenant/domains~
- ~:session/realm~ — Realm data or anomaly from domain resolution

** Host header handling
Parse the Host header to extract hostname and optional port:
- No port → lookup hostname as-is
- Port 443 + HTTPS → strip port, lookup hostname
- Port 80 + HTTP → strip port, lookup hostname
- Non-default port → anomaly (~::anomaly/incorrect~, "unexpected port")

** Flow
#+begin_example
Request
  │
  ▼
wrap-realm
  ├─ Extract Host header
  ├─ Parse hostname (validate port if present)
  ├─ Query: find realm by domain
  └─ Inject :session/realm (realm map OR anomaly)
  │
  ▼
wrap-require-realm (guard)
  ├─ Realm? → continue to handler
  └─ Anomaly? → short-circuit with error page
      ├─ ::not-found → 404 page
      └─ ::incorrect → 400 page
#+end_example

** Error pages
Full HTML pages with layout and bits.js loaded. Not plain text responses.
Different messages for different anomaly categories.
