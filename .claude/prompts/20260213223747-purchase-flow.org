* Goal
Modal checkout flow for purchasing individual posts. Supports payment method
selection (card, Monero), processing state, success confirmation, and state
transitions via SSE morphing.

* Reference
~.claude/references/purchase-flow-reference.html~ — Lines 617-1409

* Flow States
| State      | Trigger             | UI                             |
|------------+---------------------+--------------------------------|
| ~closed~   | (initial)           | No modal visible               |
| ~select~   | begin purchase      | Item summary + payment methods |
| ~process~  | confirm purchase    | Spinner, "Processing payment…" |
| ~success~  | Payment webhook     | Checkmark, "Purchase complete" |
| ~closed~   | dismiss             | Modal removed, post unlocked   |

** checkout-modal
Renders backdrop and delegates to state-specific content component.

** payment-option
Uses ~bits.form/form~ for CSRF. Displays radio-style selection.

** checkout-select
Item summary, payment method selection, total, terms, and confirm button.
All forms use ~bits.form/form~.

** checkout-processing
Centered spinner with "Processing payment…" text.

** checkout-success
Checkmark icon, "Purchase complete" message, download button.

* Actions
*Note: Action names are tentative — confirm during implementation.*

| Action                 | Parameters      | Effect                        |
|------------------------+-----------------+-------------------------------|
| begin purchase         | ~post-id~       | Morph in modal (select state) |
| select payment method  | ~post-id method~| Update selected method        |
| confirm purchase       | ~post-id method~| Morph to processing, initiate |
| cancel purchase        | ~post-id~       | Remove modal                  |
| dismiss success        | ~post-id~       | Remove modal, unlock post     |
| download               | ~post-id~       | Generate signed download URL  |

* SSE State Transitions
1. Begin → Server morphs modal into page (state=select)
2. Select method → Server morphs modal with new selection
3. Confirm → Server morphs modal (state=processing), initiates payment
4. Payment webhook → Server morphs modal (state=success)
5. Dismiss → Server removes modal, morphs post to owned state
